WorkFlow MVP 시연 가이드

작성일: 2026-02-18 | 작성자: 김지훈 (시스템관리자/개발자)

시연 예상 시간: 약 20~25분

참석자: 황일순 대표 (보트랜스), 승근수 대표 (태경홀딩스), 센터장 (의왕/서이천), 김지훈 (개발자)
장소: 의왕센터 또는 서이천센터 (미정)

역할 배정표

사람 기기 접속 역할 역할 설명

(개발자)		 노트북 (크롬) 관리자웹 시스템관리자 전체 시연 진행 + 관리 기능 데모 (모든 메뉴 접근)
김지훈
(개발자)		 스마트폰 근로자앱 근로자 출근 / 조퇴 / 퇴근 실시연
승근수
(태경 대표)		 스마트폰 근로자앱 근로자 출근 / 퇴근 실시연 (김지훈과 동시 출근)
황일순
(보트랜스 대표)		 노트북 or 태블릿 관리자웹 대표이사 전체 센터 대시보드 확인 (계정관리 제외)
센터장
(의왕/서이천)		 노트북 or 태블릿 관리자웹 센터장 본인 센터만 보이는 대시보드 모니터링
핵심 포인트: 김지훈+승근수가 스마트폰으로 출근을 찍으면 → 황대표+센터장 노트북에서 실시간으로 반영되는 것을 즉시 확인. 이것이 시연의 최대 임팩트!

사전 준비 (시연 전날까지)

항목내용
시연 센터 확정의왕 or 서이천 → Supabase sites 테이블에 해당 센터 GPS 좌표 정확히 등록
스마트폰 2대APK 설치 — 김지훈 폰 + 승근수 대표 폰 (사전에 설치 완료)
노트북/태블릿 3대김지훈(시스템관리자), 황대표(대표이사), 센터장(센터장) 각각 관리자웹 접속
계정 세팅① 김지훈: 시스템관리자(웹) + 근로자(앱) 계정 2개
② 승근수: 근로자(앱) 계정 1개
③ 황일순: 대표이사(웹) 계정 1개
④ 센터장: 센터장(웹) 계정 1개
캘린더 일정시연 당일 "MVP 시연" 일정 미리 등록 (캘린더 기능 자연스럽게 노출)
더미 데이터지난 1주일치 출퇴근 기록 + 급여 데이터 입력 (화면이 풍성해 보임)
인터넷현장 Wi-Fi 확인 + 김지훈 모바일 핫스팟 백업 준비

시연 시나리오

0 사전 세팅 (시연 시작 전 3분)

"시작하기 전에 각자 로그인 먼저 해놓겠습니다"

순서행동포인트
1김지훈: 노트북에서 관리자웹 시스템관리자로 로그인전체 메뉴 확인 가능 상태
2황일순: 노트북/태블릿에서 관리자웹 대표이사로 로그인"이 화면이 대표님이 보실 관리자 화면입니다"
3센터장: 노트북/태블릿에서 관리자웹 센터장으로 로그인"센터장님은 본인 센터만 보입니다"
4김지훈 + 승근수: 스마트폰에서 근로자앱 로그인"근로자가 사용하는 앱입니다"

1 출근 시연 (5분) — 현장감

"지금부터 실제로 출근을 찍어보겠습니다"

순서행동포인트
1김지훈이 근로자앱에서 출근 버튼 터치 → GPS 검증 → 성공"센터 반경 내에서만 출근 가능"
2승근수 대표도 함께 출근 버튼 터치 → 성공"동시에 여러 명이 출근 가능"
3황대표+센터장 노트북 새로고침 → 출퇴근 현황에 2명 즉시 반영"지금 보시는 것처럼 실시간으로 반영됩니다"
4(사전 준비) 센터 밖에서 출근 시도한 실패 스크린샷 보여주기"GPS로 위치를 검증하기 때문에 대리출근이 불가능합니다"
핵심 멘트: "기존 수기/엑셀 관리 대비 GPS로 위치가 검증되고, 관리자 화면에 실시간으로 반영됩니다"

2 관리자웹 대시보드 설명 (5분) — 황대표+센터장에게

"지금 보고 계신 화면을 설명드리겠습니다"

순서행동포인트
1김지훈 노트북에서 캘린더 일정 클릭 → 오늘 "MVP 시연" 일정 보여주기"일정관리 기능이 내장되어 있습니다"
2요약카드 — 전체직원, 출근, 미출근 인원수 확인"한눈에 현황 파악 가능"
3출퇴근 현황 테이블에서 이름 클릭 → 근로자 상세"개인별 정보를 바로 확인 가능"
4센터장 화면과 비교 → "본인 센터만 보입니다"권한별 화면 차이를 직접 눈으로 확인

3 조퇴 시연 (3분)

"조퇴 상황도 보여드리겠습니다"

순서행동포인트
1김지훈 근로자앱에서 조퇴 버튼 → 사유 입력 ("병원 방문")"조퇴 사유가 기록으로 남습니다"
2황대표+센터장 관리자웹 새로고침 → 상태 "조퇴" 뱃지 + 비고에 사유 표시"관리자 화면에서 바로 확인 가능합니다"

4 퇴근 시연 (2분)

순서행동포인트
1승근수 대표 근로자앱에서 퇴근 버튼 터치GPS 검증 후 퇴근 처리
2관리자웹에서 근무시간 자동 계산 확인"출퇴근 시간 기반으로 자동 계산됩니다"

5 관리 기능 데모 (7분) — 김지훈 노트북에서 진행

"축적된 데이터로 할 수 있는 것들을 보여드리겠습니다"

순서행동포인트
1근태기록 탭 → 날짜 범위 조회 (더미 데이터)"원하는 기간별 조회 가능"
2근태 수정/삭제 (시스템관리자 권한)"실수나 예외 상황은 관리자가 보정 가능"
3급여관리 탭 → 급여 목록 확인"근태 데이터 기반 급여 자동 산출"
4급여 확정 → 엑셀 다운로드"확정 후 엑셀로 바로 추출, 기존 업무와 100% 호환"
5설정 → 공지사항 등록 → 근로자앱에서 확인"공지사항이 근로자 앱에 실시간 전달"
6계정관리 → 새 관리자 계정 생성 (이메일+비밀번호)"센터장/대표 계정을 바로 발급 가능" (시스템관리자 전용)
TIP: 엑셀 다운로드는 대표님들이 가장 관심 가지는 기능입니다. "기존 엑셀 업무와 100% 호환"을 강조하세요.

6 권한 체계 비교 (3분) — 화면 나란히 보여주기

"각 역할마다 볼 수 있는 화면이 다릅니다"

역할접속자볼 수 있는 것현장에서 확인
시스템관리자 김지훈 노트북 전체 센터, 모든 메뉴 (계정관리 포함) 6개 메뉴 모두 보이는 것 확인
대표이사 황일순 노트북 전체 센터, 계정관리 제외 계정관리 메뉴가 없는 것 확인
센터장 센터장 노트북 본인 센터만 다른 센터 데이터 안 보이는 것 확인
근로자 김지훈/승근수 스마트폰 본인 출퇴근/급여만 앱에서 본인 기록만 보이는 것 확인

시연 팁

1. GPS 실패 시연은 스크린샷으로 준비
현장에서 센터 밖으로 나가기 어려우므로, 사전에 센터 밖에서 실패한 화면을 스크린샷으로 준비해 보여주세요.
2. 더미 데이터를 충분히 넣으세요
지난 1주일치 출퇴근 + 급여 데이터가 있어야 근태기록/급여 탭이 실제 운영처럼 보입니다.
3. "앱 출근 → 웹 즉시 확인" 흐름이 핵심
김지훈+승근수가 앱으로 출근 → 황대표+센터장이 웹에서 바로 확인하는 순간이 가장 큰 임팩트입니다.
4. 인터넷 백업 필수
물류센터 Wi-Fi가 불안정할 수 있습니다. 김지훈 폰 핫스팟을 반드시 준비하세요.
5. 승근수 대표 폰에 APK 사전 설치
시연 당일 설치하면 시간이 걸립니다. 반드시 전날까지 설치 + 로그인 테스트 완료해두세요.
