# v1.12: 테이블 높이 데이터 양에 맞게 자동 조절

## 작업 일자
2026-02-10

## 작업 내용

### 문제
- `StickyHeaderTable.wrapWithCard`가 `Expanded` 부모의 전체 높이를 사용
- 데이터가 적은 경우(예: 근로자 6명) 테이블 카드가 화면 하단까지 빈 공간으로 확장
- 특히 근로자관리 탭에서 인디고 헤더 배경이 빈 영역까지 펼쳐지는 현상

### 해결
- `wrapWithCard`에서 데이터 행 수 기반 높이 계산:
  - `contentHeight = 46 (헤더) + rowCount * 43 (행) + 2 (테두리)`
  - `effectiveHeight = min(contentHeight, constraints.maxHeight)`
- 데이터 적으면 카드가 딱 맞게 축소, 많으면 남은 공간 채우고 스크롤

### 적용 범위
- 모든 테이블 화면에 공통 적용 (대시보드, 근로자관리, 근태기록, 급여관리, 근로자 상세)
