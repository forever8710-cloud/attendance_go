# v1.7 — UI 브랜딩, 메뉴 구조 개선, 센터 접근 제어

## 변경 요약
브랜딩 변경, 메뉴 구조 개선, 출퇴근 현황 화면 개선, 역할별 센터 접근 제어를 구현하였습니다.

## 주요 변경 사항

### 1. 브랜딩 변경
- 출퇴근GO → **Workflow**
- 출퇴근GO Admin → **Workflow by TKholdings**
- 로그인 화면, 사이드 메뉴, 상단 헤더 모두 적용

### 2. 메뉴 구조 개선
| 기존 | 변경 후 |
|------|---------|
| 홈 | 출퇴근 현황 (시계 아이콘) |
| 근로자 관리 | 근로자 관리 |
| 근태 기록 | └ 근태 기록 (하위메뉴, 들여쓰기) |
| 급여 관리 | └ 급여 관리 (하위메뉴, 들여쓰기) |
| 계정 관리 | └ 계정 관리 (하위메뉴, 들여쓰기) |
| 설정 | 설정 (맨 아래로 이동) |

- NavigationRail → 커스텀 ListView 메뉴로 변경
- 하위메뉴는 좌측 패딩(40px)으로 시각적 구분

### 3. 출퇴근 현황 화면 개선
- "홈" 글씨 제거, 날짜만 크게 표시 (24px, bold)
- 모든 테이블 컬럼 정렬 가능 (No., 사업장, 성명, 직위, 직무, 출근, 퇴근, 근무시간, 상태, 비고)

### 4. 역할별 센터 접근 제어
| 역할 | 센터 접근 |
|------|----------|
| 시스템관리자 | 전체 센터 선택 가능 |
| 대표이사 | 전체 센터 선택 가능 |
| 센터장 | 본인 센터만 (드롭다운 비활성화) |

### 5. 센터 목록 수정
- 기존: 전체, 서이천, 의왕, 부평, 남사
- 변경: 전체, 서이천, 안성, 의왕, 부평 (남사 제거, 안성 추가)

### 역할별 센터 접근 (dashboard_screen.dart)
```dart
List<String> get _availableCenters {
  if (canAccessAllSites(widget.role)) {
    return _allCenters; // ['전체', '서이천', '안성', '의왕', '부평']
  } else {
    // 센터장은 본인 센터만
    final mySite = _getSiteName(widget.userSiteId);
    return [mySite];
  }
}
```

## 테스트 방법
1. `flutter run -d chrome`
2. 시스템관리자로 데모 로그인 → 센터 드롭다운에서 전체/각 센터 선택 가능
3. 센터장으로 데모 로그인 → 서이천만 표시, 변경 불가
4. 메뉴에서 하위메뉴 들여쓰기 확인
5. 출퇴근 현황 테이블에서 각 컬럼 클릭하여 정렬 확인
