## 1. 문서 개요

### 1.1 문서 목적
본 문서는 WorkFlow 제품의 요구사항을 구체적으로 명세하여, 개발팀, 디자인팀, 사업팀이 동일한 목표와 범위를 공유하도록 한다.

### 1.2 프로젝트 배경
물류센터는 대한민국 경제의 핵심 인프라이지만, 현장 근태 관리는 여전히 수기 작성과 엑셀 집계에 의존하고 있다. 이로 인해 급여 오류, 분쟁, 관리자의 과중한 업무 부담이 발생한다. WorkFlow는 이러한 문제를 GPS 기반 디지털 출퇴근 시스템으로 해결한다.

### 1.3 제품 범주
- **카테고리:** HR Tech / Workforce Management
- **세부 분야:** 근태관리(Time & Attendance), 급여관리(Payroll)
- **서비스 형태:** B2B SaaS (기업용 서비스형 소프트웨어)
- **배포 플랫폼:** Web (관리자용), iOS/Android (근로자용)

## 2. 프로젝트 비전 및 목표

### 2.1 비전
> "물류 현장 근로자들의 출퇴근과 급여를 투명하게,  
> 관리자의 HR 업무를 단순하게"

### 2.2 제품 목표

#### 2.2.1 근로자 관점
- 스마트폰 하나로 출퇴근 기록 완료 (5초 이내)
- 실시간으로 자신의 근무시간과 예상 급여 확인
- 관리자에게 문의 없이 투명한 정보 접근

#### 2.2.2 관리자 관점
- 50명 근로자의 출퇴근 현황 실시간 모니터링
- 엑셀 작업 없이 클릭 몇 번으로 급여대장 생성
- 급여 계산 오류 최소화 (자동 계산)

#### 2.2.3 비즈니스 목표
- 소규모 물류센터(근로자 50명 이하) 타겟
- E-COUNT ERP 대비 1/10 복잡도, 1/5 가격
- 매뉴얼 없이 5분 안에 사용 가능한 직관성

## 3. 사용자 페르소나

### 3.1 Primary Persona: 김하역 (근로자)

**인구통계:**
- 나이: 52세
- 직업: 물류센터 하역 작업자 (일용직)
- 근무지: 서이천 물류센터
- IT 숙련도: 낮음 (카카오톡, 네이버 정도만 사용)

**행동 패턴:**
- 아침 7시 출근, 오후 5시 퇴근
- 스마트폰은 주로 전화와 메시지용
- 복잡한 앱은 사용 포기함

**니즈:**
- "내가 몇 시간 일했는지, 얼마 받는지만 알고 싶다"
- "버튼 하나로 출퇴근 찍고 끝났으면 좋겠다"
- "급여가 맞는지 확인하려고 매번 관리자에게 전화하기 귀찮다"

**페인 포인트:**
- 수기 출퇴근부에 서명하는 게 번거롭고, 나중에 분실되기도 함
- 급여일에 금액이 예상과 다르면 불안함
- IT 용어나 복잡한 UI는 이해가 안 됨

### 3.2 Secondary Persona: 박관리 (현장 관리자)

**인구통계:**
- 나이: 38세
- 직업: 부평 물류센터 현장 관리자
- 관리 인원: 근로자 50명
- IT 숙련도: 중간 (엑셀 능숙, ERP 경험 있음)

**행동 패턴:**
- 아침 6시 출근, 출근 현황 확인
- 매일 근태 관리, 월말에 급여 집계
- PC 또는 태블릿 사용

**니즈:**
- "오늘 누가 결근했는지 한눈에 보고 싶다"
- "급여 계산 실수 없이 자동으로 처리되었으면"
- "엑셀에 데이터 옮기는 시간을 줄이고 싶다"

**페인 포인트:**
- 50명 출퇴근 수기 집계 → 반나절 소요
- 파트별 시급 다름 → 계산 실수 빈번
- 근로자들의 급여 문의 전화 대응에 시간 뺏김
- 공휴일/심야 수당 계산 복잡 → 오류 발생

**인구통계:**

**니즈:**

## 4. 문제 정의

### 4.1 현재 상황 (As-Is)

**근로자:**
- 수기 출퇴근부 작성 → 분실, 위조 가능
- 자신의 근무시간/급여 확인 어려움
- 급여 문의 시 관리자 의존

**관리자:**
- 100명 출퇴근 수기 집계 → 엑셀 작업 반나절
- 근로자별 파트/시급 상이 → 계산 실수
- 공휴일/심야/연장 수당 계산 복잡 → 급여 오류
- 실시간 근태 현황 파악 불가

**공통:**
- 데이터 분산 (엑셀 파일 여러 개)
- 투명성 부족 → 급여 분쟁
- 법적 증빙 자료 미흡

### 4.2 목표 상황 (To-Be)

**근로자:**
- 앱에서 버튼 한 번으로 GPS 기반 출퇴근 (5초)
- 실시간 근무시간/예상급여 확인
- 관리자 문의 없이 자가 확인

**관리자:**
- 실시간 출퇴근 현황 대시보드 (출근, 퇴근, 지각, 조퇴, 외출중, 결근, 정규휴무)
- 클릭 몇 번으로 급여대장 자동 생성
- 급여 계산 오류 최소화

**공통:**
- 단일 플랫폼에 모든 데이터 통합
- 투명한 정보 공유 → 분쟁 감소
- GPS/시간 기록으로 법적 증빙 확보

## 5. 솔루션 개요

### 5.1 제품 컨셉
WorkFlow는 **"최소 조작으로 끝나는 근태/급여 관리"**를 지향한다.

1. **Step 1:** 근로자가 사업장 500m 반경에 진입하면 자동 출근 기록 (GPS 기반)
2. **Step 2:** 점심시간은 관리자 설정 규칙에 따라 자동 차감 (근로자 조작 불필요)
3. **Step 3:** 정규시간 이후 500m 이탈 시 자동 퇴근 기록 / 조퇴 시 앱에서 조퇴 버튼
4. **Step 4:** 관리자가 급여대장을 확인한다 (자동 생성, 점심시간 차감 반영)

### 5.2 핵심 가치 제안

| 이해관계자 | 가치 제안 |
|----------|---------|
| 근로자 | "5초 출퇴근, 투명한 근무내역, 급여 확인" |
| 관리자 | "엑셀 작업 제로, 실시간 모니터링" |
| 사업장 대표 | "급여 분쟁 감소, 비용 절감" |

### 5.3 차별화 요소

**vs. E-COUNT ERP:**
- ✅ 10배 단순한 UI (근태/급여만 집중)
- ✅ 5배 저렴한 가격
- ✅ 매뉴얼 불필요 (직관적 UX)

**vs. 수기/엑셀:**
- ✅ 자동화 (계산 오류 제로)
- ✅ 실시간 (즉시 확인 가능)
- ✅ 투명성 (근로자도 접근)

**vs. 타 근태 앱:**
- ✅ GPS 기반 위치 검증
- ✅ 물류 현장 특화 (파트별 시급 관리)
- ✅ 급여대장 자동 생성 (엑셀 내보내기)

## 6. 핵심 기능 명세

### 6.1 MVP 기능 (Phase 1 - 1개월)

#### 6.1.1 인증 및 사용자 관리

- **근로자 계정 생성:** 관리자가 근로자 기본정보 등록
  - 이름, 전화번호
  - 소속 회사: 보트랜스(원청, BT) / 태경홀딩스(하청, TK) — 드롭다운
  - 소속 센터: 서이천(IC) / 안성(AS) / 의왕(UW) / 부평(BP) — 드롭다운
  - 파트: 지게차 / 지게차(야간) / 피커 / 피커(야간) / 검수 / 사무 — 드롭다운
  - **사번 자동생성:** `[회사코드]-[센터코드][순번3자리]` (예: BT-IC001, TK-AS002)
- **근로자 로그인 (SNS 기반):**
  - **① 카카오 로그인 (Primary):** 대부분의 한국인 근로자 대상
    - 카카오 비즈앱 연동 → 전화번호 자동 수집 (동의 시)
    - 수집 정보: 닉네임, 프로필 이미지, 이메일(선택), 전화번호(비즈앱)
    - 관리자 등록 전화번호와 자동 매칭 → 계정 연결
  - **② 구글 로그인 (Secondary):** 외국인 근로자 또는 카카오 미사용자
    - 구글 계정 연동
    - 수집 정보: 이름, 이메일, 프로필 이미지
    - 전화번호 미제공 → 수동 입력 + SMS OTP 인증으로 본인확인
    - 인증된 전화번호로 관리자 등록 데이터와 매칭
  - **③ SMS OTP (Fallback):** SNS 계정 없는 근로자
    - 전화번호 + 6자리 인증번호 (기존 방식)

- **근로자 추가정보 입력 (최초 로그인 시):**
  - 주민등록번호 (또는 외국인등록번호) — 급여·원천세용
  - 주소 — **행정안전부 도로명주소 API 연동 자동검색** (juso.go.kr, 무료)
    - 동/도로명 입력 시 실시간 자동완성
    - 선택 후 상세주소 수동 입력
  - 계좌정보 (은행, 계좌번호) — 급여 지급용

- **관리자 로그인:** 이메일 + 비밀번호
- **사업장(센터) 설정:** 4개 센터 — 서이천(IC), 안성(AS), 의왕(UW), 부평(BP)
  - 센터명, GPS 좌표, 허용 반경(500m), 점심시간, 근무시간

- **법적 동의 수집 (앱 최초 실행 시):**
  - **① 위치정보 수집·이용 동의** (위치정보법 제15조, 별도 체크박스)
    - 수집·이용 목적: 출퇴근 시간 기록, 근태 관리, 급여 계산
    - 수집 항목: GPS 좌표(위도/경도), 수집 시각
    - 보유·이용 기간: 근로관계 종료 후 3년 (근로기준법)
    - 동의 거부 권리 및 거부 시 대안: 수동 출퇴근부 작성으로 대체
  - **② 개인정보 수집·이용 동의** (개인정보보호법, 별도 체크박스)
    - 수집 항목: 이름, 전화번호, 주민번호, 주소
    - 수집 목적: 급여 지급, 원천세 신고, 근로계약 관리
    - 보유 기간: 근로관계 종료 후 3년
  - 각 동의는 **별도 체크박스**로 분리 (묶어서 일괄 동의 불가)
  - 동의서 전문은 앱 내에서 열람 가능
  - 동의 거부 시에도 앱 사용 가능 (수동 출퇴근 모드로 전환)
  - **동의 철회:** 앱 설정에서 언제든 철회 가능, 철회 시 수동 모드 전환

#### 6.1.2 출퇴근 기록 (근로자 앱)

**GPS 기반 자동 감지 (하이브리드 방식):**
- 사업장 반경 **500m**을 기준으로 GPS 위치를 감지
- 출퇴근은 GPS 자동 + 수동 버튼 병행 (근로자가 선택)

- **출근 체크인:**
  - **자동 방식:** 사업장 반경 500m 진입 시 자동 출근 기록 (백그라운드 GPS)
  - **수동 방식:** "출근" 버튼 탭 → GPS 위치 검증 → 출근 기록
  - 출근 시간 기록 (초 단위)
  - 성공/실패 피드백 (시각적 + 햅틱)
  - 위치정보 동의 미동의 시 수동 출퇴근 모드로 작동 (GPS 자동 감지 비활성화)

- **퇴근 체크아웃:**
  - **자동 방식:** 정규근무 종료 이후 + 사업장 500m 이탈 시 자동 퇴근 기록
  - **수동 방식:** "퇴근" 버튼 탭 → GPS 위치 수집 → 퇴근 기록
  - 당일 근무시간 자동 계산 표시 (점심시간 차감 반영)

- **점심시간 자동 차감:**
  - 관리자가 사업장별 점심시간 규칙 설정 (예: 12:00~13:00)
  - 해당 시간대의 GPS 이탈은 사유를 묻지 않음 (자동 "점심" 처리)
  - 급여 계산 시 점심시간 자동 차감 (근로자 조작 불필요)
  - 점심시간 규칙은 관리자 웹 설정 화면에서 관리

- **조퇴 (MVP):**
  - 근로자 앱에 "조퇴" 버튼 제공
  - 조퇴 버튼 탭 → 조퇴 사유 선택 (병원/개인사유/기타) → 확인
  - 조퇴 시간 기록, 퇴근으로 확정 처리
  - 관리자 대시보드에 조퇴 상태 표시

- **오늘 근무 요약:**
  - 출근 시간, 퇴근 시간 (또는 조퇴 시간)
  - 총 근무시간 (점심시간 차감 반영)
  - 근무 상태 (근무중 / 퇴근 / 조퇴)
  - (향후: 예상 급여)

#### 6.1.3 근태 현황 모니터링 (관리자 웹)
- **실시간 대시보드:**
  - 오늘 출근 현황 (출근 N명 / 전체 M명)
  - 미출근 근로자 목록
  - 출근 중 근로자 목록 (현재 근무 중)
  - 조퇴 근로자 목록 (조퇴 사유 포함)
  - 퇴근 완료 근로자 목록

- **출퇴근 기록 조회:**
  - 날짜별 필터
  - 근로자별 필터
  - 상세 기록 (출근/퇴근 시간, GPS 좌표, 근무시간)

#### 6.1.4 급여 관리
- **파트 관리:**
  - 6개 파트: 지게차, 지게차(야간), 피커, 피커(야간), 검수, 사무
  - 파트별 시급/일급 설정
  - 파트별 근로자 할당

- **월간 근무 집계:**
  - 근로자별 총 근무시간 (점심시간 자동 차감 반영)
  - 출근일수, 조퇴일수, 결근일수, 휴무일수, 연장시간
  - 근무파트별 시급 적용한 기본 급여 계산

- **급여대장 생성:**
  - 근로자명, 근무파트, 출근일수, 총 근무시간, 급여 총액
  - 엑셀 내보내기 (.xlsx)

#### 6.1.5 근로자 관리 (관리자 웹)
- **근로자 등록:**
  - 관리자 입력: 이름, 전화번호, 소속회사(BT/TK), 소속센터, 파트
  - 사번 자동생성: `[회사]-[센터][순번]` (예: BT-IC001)
  - 등록 완료 시 근로자에게 앱 설치 안내 (문자/카카오톡)
- **근로자 추가정보:** 근로자가 앱 최초 로그인 시 직접 입력 (주민번호, 주소, 계좌)
- **근로자 수정:** 소속회사, 센터, 파트 변경, 정보 업데이트
- **근로자 삭제:** 퇴사 처리
- **근로자 목록:** 검색, 필터(소속회사별/센터별/파트별), 정렬

## 7. 사용자 시나리오

### 7.1 시나리오 A: 근로자의 하루

**배경:**  
김하역(남, 32세)은 서이천 물류센터에서 하역 작업을 하는 일용직 근로자다. 오늘도 아침 7시에 출근한다.

**시나리오:**

**07:00 - 출근 (자동)**
1. 김하역은 물류센터 500m 반경에 도착한다.
2. 스마트폰 백그라운드에서 GPS가 사업장 진입을 감지한다.
3. 푸시 알림: "출근이 자동 기록되었습니다 ✓ (07:00:23)"
4. 스마트폰을 주머니에 넣고 작업을 시작한다.
   *(또는 수동으로 앱을 열어 "출근" 버튼을 탭할 수도 있다)*

**12:00 - 점심시간 (자동 처리)**
- 점심 먹으러 센터 밖으로 나가도 별도 조작 불필요
- 관리자가 설정한 점심시간 규칙 (12:00~13:00)에 따라 자동 차감
- 앱을 열면 "오늘 근무: 5시간 0분 (점심 제외)" 확인 가능

**15:00 - 조퇴 (해당일만)**
- (조퇴가 필요한 날) 앱을 열어 "조퇴" 버튼을 누른다.
- 사유 선택: "병원" → 확인
- "조퇴가 기록되었습니다 ✓ (15:00:12)"
- "오늘 근무: 7시간 0분 (점심 제외)" 표시

**17:00 - 퇴근 (자동)**
1. (정상 퇴근일) 작업을 마치고 물류센터를 떠난다.
2. 사업장 500m 이탈 + 정규시간(17:00) 이후 → 자동 퇴근 기록
3. 푸시 알림: "퇴근이 자동 기록되었습니다 ✓ (17:05:30)"
4. "오늘 근무: 9시간 0분 (점심 1시간 차감)" 표시
5. (향후) "예상 급여: 124,200원" 표시
   *(또는 수동으로 앱을 열어 "퇴근" 버튼을 탭할 수도 있다)*

**효과:**
- 출퇴근 기록에 **조작 0초** (자동 감지) 또는 5초 (수동 버튼)
- 점심시간 별도 조작 불필요 (자동 차감)
- 조퇴 시에만 최소한의 입력 (버튼 1회 + 사유 선택)
- 수기 출퇴근부 불필요
- 자신의 근무시간 투명하게 확인 (점심 차감 반영)

### 7.2 시나리오 B: 관리자의 아침

**배경:**  
박관리(38세)는 부평 물류센터의 현장 관리자다. 50명의 근로자를 관리하며, 매일 아침 출근 현황을 확인해야 한다.

**시나리오:**

**06:30 - 출근 현황 확인**
1. 사무실 PC에서 WorkFlow 웹 대시보드 접속
2. 로그인 (이메일 + 비밀번호)
3. 메인 화면에 큰 숫자가 보인다:
   - "오늘 출근: 3명 / 50명"
4. 시간이 지나면서 숫자가 실시간으로 증가한다.
5. 09:00에 "47명 / 50명" 확인
6. "미출근 3명" 클릭하면 목록 표시:
   - 김하역, 이포장, 박운송
7. 김하역에게 전화로 확인 → "늦잠 자서 9시 30분에 도착 예정"

**효과:**
- 수기 출퇴근부 확인 불필요
- 실시간으로 누가 안 왔는지 파악
- 전화 확인 시간 단축

### 7.3 시나리오 C: 월말 급여 처리

**배경:**  
박관리는 매달 말일에 급여대장을 작성해야 한다. 기존에는 수기 출퇴근부를 엑셀로 옮기는 데 반나절이 걸렸다.

**시나리오:**

**기존 방식 (As-Is):**
1. 50명의 수기 출퇴근부 수거 (일부 분실)
2. 날짜별로 출근 체크 확인
3. 엑셀에 일일이 입력 (3시간 소요)
4. 파트별 시급 확인하며 급여 계산
5. 계산기로 검산 (실수 발생)
6. 완성된 엑셀 파일 저장

**WorkFlow 방식 (To-Be):**
1. 웹 대시보드 "급여" 메뉴 클릭
2. "서이천센터, 2026년 1월" 선택
3. "급여대장 생성" 버튼 클릭 (1초)
4. 자동 생성된 급여대장 확인:
   - 근로자명, 파트, 출근일수, 총 근무시간, 급여
5. 이상 항목 검토 (5분)
6. "엑셀 내보내기" 클릭
7. 다운로드된 파일을 회계팀에 전달

**효과:**
- 작업 시간: 3시간 → 10분 (95% 단축)
- 계산 오류 제로
- 급여 관련 근로자 문의 감소

**Must Have (MVP 필수):**
- 출퇴근 체크인/체크아웃 (GPS 500m 자동 감지 + 수동 버튼)

- 점심시간 규칙 설정 및 자동 차감

- 월간 근무시간 집계 (점심 차감 반영)
- 기본 급여대장, 급여명세서 생성 및 엑셀 내보내기

**Should Have (MVP 직후):**
- 외출/복귀 GPS 추적 (점심 외 이탈 시 사유 태그)
- 근태 수정 승인 워크플로우
- 푸시 알림 (GPS 이탈 알림 포함)
- 공휴일 자동 판별

- 복잡한 수당 자동 계산
- 고급 리포트 및 분석
- 카카오톡 알림톡 연동

**Won't Have (범위 제외):**
- 급여 지급 기능 (은행 연동)
- 채용 관리
- 근무 스케줄링 (교대 근무 등)
