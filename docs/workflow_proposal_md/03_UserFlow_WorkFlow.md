## 1. 문서 개요

### 1.1 문서 목적
본 문서는 WorkFlow 시스템의 주요 사용자 행동 흐름을 시각화하여, UX/UI 디자인과 개발 구현의 기준을 제공한다.

### 1.2 User Flow 범위
- 근로자의 출퇴근 체크인/체크아웃 (GPS 500m 자동 감지 + 수동 버튼)
- 근로자의 조퇴 처리
- 점심시간 자동 차감 (관리자 규칙 기반)
- 관리자의 근태 모니터링 및 급여 처리
- 예외 상황 처리 (GPS 실패, 네트워크 오류 등)

### 1.3 표기 규칙
- 📱 = 모바일 앱 (근로자)
- 💻 = 웹 대시보드 (관리자)
- 🔧 = 시스템 처리
- ❌ = 오류 처리
- ✅ = 성공 처리

## 2. 근로자 User Flow

### 2.1 회원가입 및 로그인 (SNS 기반)

```mermaid
flowchart TD
    A[앱 최초 실행] --> B{로그인 필요}
    B -->|No| M[메인 화면 진입]
    B -->|Yes| C[로그인 방식 선택 화면]

    C -->|카카오 로그인| D1[카카오 SDK 로그인]
    D1 --> D2{카카오 인증 성공?}
    D2 -->|No| C
    D2 -->|Yes| D3[전화번호 자동 수집 - 비즈앱]
    D3 --> H[DB에서 전화번호로 근로자 조회]

    C -->|구글 로그인| E1[구글 Sign-In]
    E1 --> E2{구글 인증 성공?}
    E2 -->|No| C
    E2 -->|Yes| E3[전화번호 수동 입력]
    E3 --> E4[SMS OTP 발송]
    E4 --> E5[인증번호 입력]
    E5 --> E6{OTP 검증 성공?}
    E6 -->|No| E5
    E6 -->|Yes| H

    C -->|SMS 인증| F1[전화번호 입력]
    F1 --> F2[SMS OTP 발송]
    F2 --> F3[인증번호 입력]
    F3 --> F4{OTP 검증 성공?}
    F4 -->|No| F3
    F4 -->|Yes| H

    H --> I{등록된 근로자?}
    I -->|No| J[관리자에게 등록 요청 안내]
    J --> K[종료]
    I -->|Yes| P{추가정보 입력 완료?}
    P -->|No| PA[추가정보 입력 폼]
    PA --> PB[주민등록번호 or 외국인등록번호]
    PB --> PC[주소 검색 - 도로명주소 API]
    PC --> PD[상세주소 입력]
    PD --> PE[계좌정보 입력]
    PE --> PF[저장]
    PF --> L{법적 동의 완료?}
    P -->|Yes| L
    L -->|No| LA[동의 화면 표시]
    LA --> LB[① 위치정보 수집 동의 체크박스]
    LB --> LC[② 개인정보 수집 동의 체크박스]
    LC --> LD{동의 선택}
    LD -->|전체 동의| LE[GPS 자동 모드 활성화]
    LD -->|위치정보 거부| LF[수동 출퇴근 모드]
    LE --> M
    LF --> M
    L -->|Yes| M
```

**상세 설명:**

1. **앱 최초 실행**
   - 스플래시 화면 표시 (1초)
   - 로그인 상태 확인

2. **로그인 방식 선택 화면**
   - **카카오 로그인** 버튼 (노란색, 최상단 — Primary)
   - **구글 로그인** 버튼 (흰색 테두리)
   - **SMS 인증** 텍스트 링크 (하단, Fallback)

3. **카카오 로그인 (Primary)**
   - 카카오 SDK 호출 → 카카오톡 앱 또는 웹 인증
   - 비즈앱 연동 시 전화번호 자동 수집 (사용자 동의 포함)
   - 수집 정보: 닉네임, 프로필 이미지, 이메일(선택), 전화번호
   - 자동 수집된 전화번호로 workers 테이블 매칭

4. **구글 로그인 (외국인 근로자)**
   - 구글 Sign-In 호출
   - 수집 정보: 이름, 이메일, 프로필 이미지
   - 전화번호 미제공 → 수동 입력 화면
   - 입력된 전화번호로 SMS OTP 발송 → 본인확인
   - 인증 완료 후 workers 테이블 매칭

5. **SMS OTP (Fallback)**
   - 전화번호 입력 → 6자리 인증번호 발송
   - 유효 시간: 3분, 재발송 가능 (1분 후)
   - 기존 방식과 동일

6. **사용자 매칭**
   - 인증된 전화번호로 workers 테이블 조회
   - 미등록 시 관리자 연락처 안내 (관리자가 먼저 등록해야 함)

7. **추가정보 입력** (최초 로그인 시 1회)
   - 주민등록번호 또는 외국인등록번호 (급여·원천세용)
   - 주소 — 도로명주소 API 자동검색 (동/도로명 입력 → 실시간 결과)
   - 상세주소 수동 입력
   - 계좌정보 (은행 선택 + 계좌번호)
   - 입력 완료 시 `is_profile_complete = true`

8. **법적 동의 화면** (추가정보 입력 후 1회)
   - 위치정보 수집·이용 동의 (별도 체크박스)
     * 목적, 항목, 보유기간, 거부 권리 고지
   - 개인정보 수집·이용 동의 (별도 체크박스)
   - 동의서 전문 열람 링크 제공
   - 위치정보 동의 거부 시: 수동 출퇴근 모드 안내
   - 개인정보 동의는 필수 (미동의 시 서비스 이용 불가)
   - 동의 이력 DB 기록 (consent_logs)

### 2.2 출근 체크인 (하이브리드 방식)

#### 2.2.1 자동 출근 (GPS 감지)

```mermaid
flowchart TD
    A[백그라운드 GPS 모니터링] --> B{사업장 500m 진입?}
    B -->|No| A
    B -->|Yes| C{오늘 출근 기록 존재?}
    C -->|Yes| D[무시 - 이미 출근]
    C -->|No| E[DB에 출근 기록 저장]
    E --> F[푸시 알림: 출근 자동 기록]
    F --> G[관리자 대시보드 갱신]
```

#### 2.2.2 수동 출근 (버튼)

```mermaid
flowchart TD
    A[메인 화면] --> B[출근 버튼 탭]
    B --> C[위치 권한 확인]
    C --> D{권한 허용?}
    D -->|No| E[권한 요청 다이얼로그]
    E --> F{권한 승인?}
    F -->|No| G[권한 필요 안내]
    G --> A
    F -->|Yes| H[GPS 위치 수집]
    D -->|Yes| H

    H --> I{GPS 정확도 OK?}
    I -->|No| J[정확도 낮음 경고]
    J --> K[재시도 or 취소]
    K --> H

    I -->|Yes| L[사업장과 거리 계산]
    L --> M{반경 500m 이내?}
    M -->|No| N[위치 오류 메시지]
    N --> O[관리자 문의 안내]
    O --> A

    M -->|Yes| P[DB에 출근 기록 저장]
    P --> Q[성공 피드백]
    Q --> R[출근 시간 표시]
    R --> S[근무 중 화면]
```

**상세 설명:**

1. **메인 화면 구성**
   - 상단: 사용자 이름, 사업장명
   - 중앙: 큰 "출근" 버튼 (초록색) + "조퇴" 버튼 (주황색, 출근 후 표시)
   - 하단: 최근 출퇴근 기록
   - 자동 출근 시 "자동 출근됨" 배지 표시

2. **GPS 위치 수집**
   - Geolocator 패키지 사용
   - 정확도 < 50m 요구
   - 타임아웃: 10초
   - 백그라운드 GPS: Geofencing 사용

3. **거리 계산**
   - Haversine 공식 사용
   - 사업장 GPS 좌표와 비교
   - 허용 반경: **500m**

4. **성공 피드백**
   - 수동: 진동 (200ms) + "출근이 완료되었습니다 ✓" + 출근 시간 표시
   - 자동: 푸시 알림 "출근이 자동 기록되었습니다 (HH:MM:SS)"

### 2.3 퇴근 체크아웃 (하이브리드 방식)

#### 2.3.1 자동 퇴근 (GPS 감지)

```mermaid
flowchart TD
    A[백그라운드 GPS 모니터링] --> B{사업장 500m 이탈?}
    B -->|No| A
    B -->|Yes| C{현재 시간 >= 정규 퇴근 시간?}
    C -->|No| D[점심시간 구간인지 확인]
    D -->|점심시간| E[무시 - 점심 자동 처리]
    D -->|점심 아님| F[향후: 외출 사유 묻기 - Post-MVP]
    C -->|Yes| G{오늘 퇴근 기록 존재?}
    G -->|Yes| H[무시 - 이미 퇴근]
    G -->|No| I[DB에 퇴근 기록 저장]
    I --> J[푸시 알림: 퇴근 자동 기록]
    J --> K[관리자 대시보드 갱신]
```

#### 2.3.2 수동 퇴근 (버튼)

```mermaid
flowchart TD
    A[근무 중 화면] --> B[퇴근 버튼 탭]
    B --> C{출근 기록 존재?}
    C -->|No| D[출근 기록 없음 오류]
    D --> E[관리자 문의 안내]
    E --> A

    C -->|Yes| F[GPS 위치 수집]
    F --> G{GPS 정확도 OK?}
    G -->|No| H[정확도 낮음 경고]
    H --> I[재시도 or 강제 퇴근]
    I -->|재시도| F
    I -->|강제 퇴근| J[관리자 승인 필요 표시]

    G -->|Yes| K[사업장과 거리 계산]
    K --> L{반경 500m 이내?}
    L -->|No| M[위치 경고 표시]
    M --> N[계속 or 취소]
    N -->|취소| A
    N -->|계속| O[관리자 승인 필요 표시]

    L -->|Yes| P[DB에 퇴근 기록 업데이트]
    P --> Q[근무시간 자동 계산 - 점심 차감]
    Q --> R[성공 피드백]
    R --> S[퇴근 완료 화면]
    S --> T[오늘 근무 요약 표시]
    T --> U[메인 화면 복귀]
```

**상세 설명:**

1. **근무 중 화면**
   - 출근 시간 표시
   - 현재 근무 시간 (실시간 카운트, 점심시간 차감 반영)
   - 퇴근 버튼 (파란색) + 조퇴 버튼 (주황색)

2. **퇴근 검증**
   - 출근 기록 존재 확인
   - GPS 위치 재수집
   - 거리 재계산 (500m)

3. **강제 퇴근 옵션**
   - GPS 오류 시 선택 가능
   - 관리자 승인 필요 표시
   - 메모 입력 가능

4. **근무 요약**
   - 출근 시간, 퇴근 시간
   - 총 근무 시간 (점심 차감 반영, HH:MM 형식)
   - 점심시간 차감 여부 표시
   - (향후) 예상 급여

### 2.4 조퇴 처리

```mermaid
flowchart TD
    A[근무 중 화면] --> B[조퇴 버튼 탭]
    B --> C{출근 기록 존재?}
    C -->|No| D[출근 기록 없음 오류]
    D --> A

    C -->|Yes| E[조퇴 확인 다이얼로그]
    E --> F[조퇴 사유 선택]
    F --> G[병원 / 개인사유 / 기타]
    G --> H{확인?}
    H -->|취소| A
    H -->|확인| I[GPS 위치 수집]
    I --> J[DB에 퇴근 기록 업데이트]
    J --> K[status = early_leave]
    K --> L[근무시간 자동 계산]
    L --> M{점심시간 전 조퇴?}
    M -->|Yes| N[점심 미차감]
    M -->|No| O[점심 차감]
    N --> P[조퇴 완료 피드백]
    O --> P
    P --> Q[오늘 근무 요약 표시]
    Q --> R[메인 화면 복귀]
```

**상세 설명:**

1. **조퇴 버튼**
   - 출근 후 표시되는 주황색 버튼
   - 퇴근 버튼 옆에 배치

2. **조퇴 사유 선택**
   - 병원 (의료 목적)
   - 개인사유 (가정 사정 등)
   - 기타 (자유 입력 가능)

3. **점심시간 차감 판정**
   - 점심시간(예: 12:00) 이전 조퇴 → 점심 미차감
   - 점심시간 이후 조퇴 → 점심 차감

4. **조퇴 완료 피드백**
   - "조퇴가 기록되었습니다 ✓"
   - 조퇴 시간, 근무 시간, 조퇴 사유 표시
   - 관리자 대시보드에 조퇴 상태 표시

### 2.5 근태 기록 조회

```mermaid
flowchart TD
    A[메인 화면] --> B[내 근태 메뉴 탭]
    B --> C[이번 달 기록 로딩]
    C --> D[캘린더 뷰 표시]
    D --> E[날짜별 출퇴근 시간]
    E --> F{특정 날짜 선택?}
    F -->|Yes| G[해당 날짜 상세 화면]
    G --> H[출근/퇴근 시간, 근무시간 표시]
    H --> I[GPS 위치 지도 표시]
    I --> J[뒤로 가기]
    J --> D
    
    F -->|No| K[월 선택 변경?]
    K -->|Yes| L[이전/다음 달 선택]
    L --> M[해당 월 기록 로딩]
    M --> D
    K -->|No| N[메인 화면 복귀]
```

**상세 설명:**

1. **캘린더 뷰**
   - 월간 캘린더 형식
   - 출근 날짜는 초록색 점 표시
   - 결근 날짜는 빨간색 점 표시

2. **상세 화면**
   - 출근/퇴근 정확한 시간
   - 총 근무 시간
   - GPS 위치 지도 표시 (작은 맵)

3. **월 통계**
   - 총 출근 일수
   - 총 근무 시간
   - (향후) 예상 급여

## 3. 관리자 User Flow

### 3.1 로그인 및 대시보드

```mermaid
flowchart TD
    A[웹 브라우저 접속] --> B[로그인 화면]
    B --> C[이메일 + 비밀번호 입력]
    C --> D{인증 성공?}
    D -->|No| E[오류 메시지 표시]
    E --> C
    
    D -->|Yes| F[사용자 권한 확인]
    F --> G{관리자 권한?}
    G -->|No| H[접근 거부 메시지]
    H --> B
    
    G -->|Yes| I[대시보드 로딩]
    I --> J[오늘 출근 현황 조회]
    J --> K[실시간 데이터 구독]
    K --> L[대시보드 표시]
    L --> M[출근 N명 / 전체 M명]
    M --> N[미출근 근로자 목록]
    N --> O[실시간 업데이트 대기]
```
